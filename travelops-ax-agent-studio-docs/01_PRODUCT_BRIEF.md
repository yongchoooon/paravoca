# 제품 기획서

## 제품명

TravelOps AX Agent Studio

## 도메인

여행 액티비티 / 관광 상품 운영 자동화

## 한 줄 설명

TravelOps AX는 여행 상품 운영자가 지역 관광 데이터를 바탕으로 "무엇을 팔 수 있을지" 판단하고, 상품 초안, 상세페이지 카피, FAQ, 리스크 검수 포인트를 한 번에 준비하도록 돕는 멀티에이전트 워크플로우 시스템입니다.

## 왜 이 프로젝트인가

이 프로젝트는 여행/액티비티 플랫폼 운영 도메인과 직접 맞닿아 있습니다. 실제 운영자가 매일 수행하는 지역 조사, 상품 후보 발굴, 카피 작성, FAQ 정리, 리스크 검수 업무를 하나의 AI workflow로 묶습니다.

포트폴리오 관점에서 보여줄 수 있는 역량은 다음과 같습니다.

- Research-to-Product: 공공 관광 데이터와 지역 수요 데이터를 실제 상품 기획 기능으로 연결합니다.
- Agent orchestration: Planner, Data, Research, Product, Marketing, QA Agent가 역할을 나누고 도구를 호출합니다.
- Human-in-the-loop: 사람 승인을 최종 저장 전 필수 단계로 둡니다.
- Evaluation: 검색 정확도, 환각률, tool call 정확도, 업무 성공률, 비용, latency를 측정합니다.
- Business impact: 운영자가 상품 기획/카피/FAQ/검수에 쓰는 시간을 줄이는 것을 수치화합니다.
- Cost governance: Gemini gateway 기반 비용 추적과 모델 라우팅으로 월 비용을 통제합니다.

## 대상 사용자

### 1. 여행 플랫폼 운영자

액티비티, 투어, 입장권, 축제 상품을 운영하는 담당자입니다. 매일 지역별 관광지와 행사 정보를 찾고, 상품 후보를 만들고, 상세페이지 카피와 FAQ를 작성하며, 과장 표현이나 날짜 오류를 검수합니다.

주요 니즈:

- 지역과 기간만 입력하면 상품 후보를 빠르게 받고 싶다.
- 출처가 있는 데이터 기반으로 기획하고 싶다.
- 상세페이지 초안을 자동으로 만들고 싶다.
- 검수 포인트를 놓치지 않고 싶다.
- 승인 전까지 외부 저장/전송이 되지 않게 하고 싶다.

### 2. 여행 상품 MD / 콘텐츠 매니저

특정 지역, 시즌, 타깃 고객에 맞춰 신규 상품을 구성하는 사람입니다.

주요 니즈:

- 외국인/가족/커플/혼행/시니어 등 타깃별 상품 아이디어가 필요하다.
- 행사/축제 날짜와 주변 관광지를 조합하고 싶다.
- 검색 키워드, SNS 문구, FAQ까지 한 번에 초안을 만들고 싶다.

### 3. 운영 리더 / CX 리더

상품 운영 품질과 리스크를 관리하는 사람입니다.

주요 니즈:

- 환각이나 출처 누락을 줄이고 싶다.
- 가격 단정, 일정 단정, 안전/법적 리스크 표현을 잡고 싶다.
- 사람이 많이 수정한 부분을 추적해 프롬프트와 데이터 품질을 개선하고 싶다.

### 4. AX 컨설팅/AI 엔지니어링 평가자

이 프로젝트를 보는 채용/포트폴리오 평가자입니다.

주요 니즈:

- 실무 프로세스를 자동화한 흔적을 보고 싶다.
- 에이전트가 어떤 도구를 왜 호출했는지 보고 싶다.
- 평가 지표와 비용 추적이 있는지 보고 싶다.
- 운영자가 실제로 승인하고 수정하는 인터페이스가 있는지 보고 싶다.

## 핵심 문제

여행 플랫폼 운영자는 상품 하나를 만들 때 다음 작업을 반복합니다.

1. 지역 관광지와 행사 정보를 검색합니다.
2. 계절, 위치, 교통, 주변 코스를 고려합니다.
3. 타깃 고객에게 맞는 상품 조합을 만듭니다.
4. 상세페이지 카피, FAQ, SNS 문구, 검색 키워드를 작성합니다.
5. 과장 표현, 날짜 오류, 출처 누락, 안전 리스크를 검수합니다.
6. 승인된 초안을 내부 시스템이나 시트에 저장합니다.

이 작업은 대부분 반정형 업무입니다. 사람이 방향을 정하고 최종 승인해야 하지만, 데이터 조회, 초안 작성, 검수 체크리스트 적용, 리포트 생성은 자동화하기 좋습니다.

## 핵심 가치 제안

### 여행 상품화

관광지, 축제, 숙박, 지역 정보를 상품 콘셉트, 코스, 운영 메모, 상세페이지 초안으로 변환합니다.

### 운영 자동화

지역, 기간, 타깃, 상품 수를 입력하면 상품 기획 리포트까지 생성합니다.

### 출처 기반 상품 기획

TourAPI와 관광 수요 데이터, 자체 수집 DB, RAG 검색 결과를 기반으로 생성 결과에 출처를 붙입니다.

### 근거 있는 생성

AI가 작성한 내용 중 근거 문서와 연결된 정보, 운영자가 최종 확인해야 하는 정보를 구분합니다.

### 운영 리스크 사전 점검

날짜 오류, 가격 단정, 과장 표현, 출처 누락, 예약 가능 여부 단정처럼 실제 판매 전 확인해야 할 항목을 QA/Compliance 단계에서 점검합니다.

### 멀티에이전트 분업

Planner Agent가 업무를 나누고, Data/Research/Product/Marketing/QA Agent가 각각 전문 작업을 수행합니다.

### 사람 승인 기반 저장

Human Approval Node를 통과해야만 결과물이 저장되거나 외부 시스템으로 전송됩니다.

### 평가와 비용 추적

업무 성공률, 검색 recall, faithfulness, tool call accuracy, cost per task, latency, human revision rate를 측정합니다.

### 저비용 API 기반 운영

Gemini API, rule-based mode, 로컬 vector index를 조합해 대규모 GPU 인프라 없이도 업무 자동화 흐름을 실행할 수 있게 설계합니다.

### Poster Studio 확장

후속 기능에서는 Run Review 결과를 활용해 여행 상품 홍보 포스터까지 생성합니다. AI가 포스터에 들어갈 정보와 시각 방향을 추천하고, 사용자가 문구와 옵션을 직접 고른 뒤 이미지 생성용 프롬프트를 확정합니다.

## 대표 사용 시나리오

사용자가 입력합니다.

```text
이번 달 부산에서 외국인 대상 액티비티 상품을 5개 기획해줘.
행사나 야간 관광 요소가 있으면 우선 반영하고,
각 상품마다 상세페이지 카피, FAQ 5개, SNS 문구 3개, 리스크 검수 결과를 포함해줘.
```

시스템은 다음 순서로 실행합니다.

1. Planner Agent가 작업을 분해합니다.
2. Data Agent가 TourAPI와 관광 수요 데이터를 조회합니다.
3. Research Agent가 지역, 계절성, 행사 여부, 타깃 적합성을 분석합니다.
4. Product Agent가 액티비티 상품 아이디어 5개를 만듭니다.
5. Marketing Agent가 상세페이지 카피, FAQ, SNS 문구, 검색 키워드를 생성합니다.
6. QA/Compliance Agent가 환각, 출처 누락, 날짜 오류, 과장 표현, 가격 단정 표현을 검수합니다.
7. UI는 최종 리포트와 검수 결과를 보여줍니다.
8. 사용자가 승인하면 draft가 approved 상태로 바뀌고 저장됩니다.

## 성공 기준

MVP 성공 기준은 다음과 같습니다.

- 사용자가 자연어 요청을 입력하면 1개 이상의 workflow run이 생성됩니다.
- workflow run에는 agent step과 tool call 로그가 남습니다.
- 최소 3개 이상의 TourAPI 도구가 실제 한국관광공사 API로 호출됩니다.
- 결과물에는 출처가 연결됩니다.
- QA/Compliance 단계가 가격 단정, 날짜 오류, 출처 누락을 감지합니다.
- 승인 전 결과는 `draft`이고, 승인 뒤 `approved`가 됩니다.
- 평가 스크립트가 최소 샘플 10개 이상을 실행하고 결과 JSON/CSV를 남깁니다.
- 프론트엔드 UI는 Mantine UI와 CSS Modules 또는 SCSS Modules 기반이며 Tailwind CSS, shadcn/ui, Bootstrap이 없습니다.

## 포트폴리오 소개 문장

관광 공공데이터와 지역 수요 데이터를 수집/정제하고, RAG 기반 상품 운영 자동화 멀티에이전트 시스템을 구현했습니다. LangGraph 기반으로 Planner, Data, Research, Product, Marketing, QA Agent를 분리했으며, 평가 파이프라인으로 검색 정확도, 환각률, 도구 호출 성공률, 업무당 비용, latency를 측정하도록 설계했습니다. React Flow와 Mantine UI 기반 운영 화면을 제공하고, Human Approval Node를 통해 승인 전 외부 저장을 차단하는 운영형 AX 시스템으로 구성했습니다.

## 공모전/포트폴리오에서 강조할 지점

이 프로젝트의 중심은 여행 상품화입니다. 공공 관광 데이터를 운영자가 검토할 수 있는 상품 초안, 상세페이지 카피, FAQ, 리스크 체크리스트, 승인 흐름으로 연결합니다.

강조할 문장:

```text
TravelOps AX는 여행 상품 운영자에게 "무엇을 팔 수 있을지"를 제안합니다.
```

평가자가 확인할 수 있는 강점:

- 여행 플랫폼 운영자가 반복하는 리서치, 문서 작성, 검수 업무를 하나의 workflow로 모델링했습니다.
- TourAPI, source document, RAG 검색 결과를 상품 기획 산출물과 연결했습니다.
- 생성 결과에 근거 문서와 확인 필요 정보를 함께 표시합니다.
- QA/Compliance Agent가 실제 판매 전 문제가 될 수 있는 표현과 정보 누락을 점검합니다.
- Human Approval과 Revision Workflow를 통해 사람이 최종 판단권을 유지합니다.
- API 사용량, latency, 예상 비용을 기록해 작은 팀에서도 운영 가능한 구조를 보여줍니다.
- 후속 Poster Studio는 상품 기획 결과를 포스터 제작으로 이어 붙여, 상품화 업무의 마지막 홍보 소재 초안까지 확장합니다.

## 후속 기능: Poster Studio

Poster Studio는 상품 기획 run에서 생성된 결과를 바탕으로 포스터 제작용 프롬프트와 이미지를 만드는 기능입니다.

목표:

- Run Review에 있는 상품명, 한 줄 소개, 핵심 가치, 코스, FAQ, 운영 주의사항을 포스터 후보 문구로 변환합니다.
- 포스터 목적, 비율, 스타일, 문구 밀도, 포함 정보, 색감, CTA를 사용자가 선택하게 합니다.
- AI가 추천한 문구와 옵션을 사용자가 직접 편집하고 삭제할 수 있게 합니다.
- 최종 프롬프트 확정 후 OpenAI Image API로 이미지를 생성합니다.
- 생성된 포스터와 프롬프트, 선택 옵션, 비용 로그를 원본 workflow run에 연결합니다.

기본 흐름:

1. 운영자가 Run Review에서 상품 1개를 선택합니다.
2. Poster Prompt Agent가 포스터 후보 문구와 디자인 방향을 제안합니다.
3. 사용자가 옵션과 문구를 검토하고 수정합니다.
4. Poster Prompt Agent가 최종 이미지 생성 프롬프트를 만듭니다.
5. Poster Image Agent가 `gpt-image-2`로 포스터 이미지를 생성합니다.
6. Poster QA 단계가 이미지 속 문구, 날짜/가격 단정, 과장 표현, 브랜드/저작권 리스크를 점검합니다.
7. 사람이 최종 포스터를 승인하거나 재생성을 요청합니다.

사용자 옵션:

- 포스터 목적: 상세페이지 대표 이미지, SNS 피드, SNS 스토리, 오프라인 A4 포스터
- 비율/크기: 1:1, 4:5, 9:16, A4 세로
- 스타일: 프리미엄 여행, 로컬 감성, 축제 홍보, 가족 친화, 액티비티 중심
- 문구 밀도: 제목만, 핵심 문구 중심, 상세 정보 포함
- 포함 정보: 상품명, 지역, 기간, 타깃, 코스, 핵심 가치, CTA, 확인 필요 문구
- 이미지 기준: AI 생성 비주얼, TourAPI 이미지 참고, 그래픽 중심 배경
