# D05. Markdown Output Contract

최종 앱 응답은 Markdown 하나만 반환한다.

## 성공 출력 구조

```text
# 여행 상품 추천
## 1. 상품명
### 상품 콘셉트
### 구성 장소와 추천 동선
### 확인된 핵심 정보
### 관련 링크
### 세일즈 포인트
### 판매/홍보 문구
### SNS 홍보안
### FAQ 초안
### 판매 실험/운영 팁
### 방문 또는 판매 전 확인할 것
### 근거
## 2. 상품명
## 3. 상품명
## 전체 판매 전략
## 꼭 확인해야 할 사항
## 사용하면 안 되는 표현
## AI 포스터 만들기
## 앞으로 가능한 것
```

## 실패 출력

Preflight 실패:

```text
# 요청을 다시 입력해 주세요
```

지역 확정 실패:

```text
# 지역을 더 구체적으로 입력해 주세요
```

데이터 부족:

```text
# 기획에 필요한 데이터가 부족합니다
```

포스터 생성 성공:

```text
# AI 포스터 생성 완료
<img src="..." />
<a href="..." target="_blank">이미지 크게 보기</a>
짧은 이미지 설명문
```

Notion 페이지 생성 성공:

```text
Notion 페이지를 만들었습니다.

[Notion에서 열기](https://www.notion.so/...)
```

## 금지

- JSON 출력 금지
- 코드블록 출력 금지
- 근거 없는 장소/가격/운영시간/행사일정 생성 금지
- `ev-001`, `ev-004` 같은 내부 evidence id 출력 금지
- 단순 관광 코스 추천으로 축약 금지
- 수집된 요금, 시설, 무료 여부, FAQ, SNS/해시태그, 세일즈 카피, 판매 실험 누락 금지
- 앞선 Agent 출력에 실제 URL이 있는 경우 관련 링크 버튼 누락 금지
- 실제 URL이 없는데 관련 링크를 추측해 생성 금지
- A14에서 포스터 이미지를 직접 생성하거나 이미지 API를 호출하는 것 금지
- A14 성공 응답에는 `AI 포스터 만들기` 안내와 예시 입력을 붙인다
- A17 포스터 생성 응답에서는 API 응답의 실제 `image_url` 없이 이미지 HTML을 만들지 않는다
- A17 포스터 생성 응답에서는 `참고 이미지`, `생성 정보`, `생성 프롬프트 요약` 같은 내부 처리 섹션을 출력하지 않는다
- A17 포스터 생성 응답에서는 `image_id`, `model`, `size`, `quality`, `latency_ms`, `provider_response_summary.endpoint` 같은 내부 생성 메타데이터를 출력하지 않는다
- A21/A24/A27은 JSON을 그대로 출력하지 않는다
- A21/A24/A27은 표, HTML 카드, HTML table 기반 시각화를 적극 시도한다
- A28R은 저장 대상 Markdown을 선택해 `title`, `markdown`, `proposal_type` payload만 만든다
- A28은 Notion 페이지 URL이 없으면 링크를 추측하지 않는다
- A28은 `page_id`, raw 응답, 처리 시간, 내부 state 이름을 출력하지 않는다
- 리비전 섹션 금지
