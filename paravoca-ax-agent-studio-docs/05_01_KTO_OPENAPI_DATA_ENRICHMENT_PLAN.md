# KTO OpenAPI 데이터 보강 계획

작성 기준일: 2026-05-06

이 문서는 현재 PARAVOCA AX Agent Studio의 TourAPI 중심 수집 흐름을 한국관광공사 OpenAPI 묶음으로 확장하기 위한 데이터 활용 계획입니다. 목적은 더 많은 API를 한 번에 호출하는 것이 아니라, 실행 요청과 현재 근거의 공백을 보고 필요한 데이터를 골라 보강하는 것입니다.

## 현재 구현 기준

현재 코드에서 실제 호출하는 관광 데이터는 `backend/app/tools/tourism.py`의 `TourApiProvider`에 집중되어 있습니다.

현재 provider base URL:

```text
https://apis.data.go.kr/B551011/KorService2
```

현재 구현된 operation:

| 현재 tool | 실제 operation | 현재 용도 |
|---|---|---|
| `tourapi_area_code` | `areaCode2` | 지역명을 지역 코드로 변환 |
| `tourapi_area_based_list` | `areaBasedList2` | 지역 기반 관광정보 후보 조회 |
| `tourapi_search_keyword` | `searchKeyword2` | 키워드 기반 관광지 후보 조회 |
| `tourapi_search_festival` | `searchFestival2` | 행사/축제 후보 조회 |
| `tourapi_search_stay` | `searchStay2` | 숙박 후보 조회 |

현재 `DataAgent`는 지역, 키워드, 행사, 숙박을 조회한 뒤 `tourism_items`와 `source_documents`에 저장하고, Chroma 검색 결과를 downstream agent에 넘깁니다.

현재 저장되는 주요 필드:

- `content_id`, `content_type`, `title`
- `region_code`, `sigungu_code`, `address`, `map_x`, `map_y`
- `tel`, `homepage`, `overview`
- `image_url`, `license_type`
- `event_start_date`, `event_end_date`
- 원본 `raw`

현재 데이터만으로 만들기 어려운 판단:

- 상품별 운영 시간, 휴무, 이용 조건, 반려동물 조건, 코스 거리, GPX 같은 실행 세부 정보
- 이미지/포스터 제작에 쓸 수 있는 사진 후보와 시각 키워드
- 방문 수요, 혼잡 가능성, 연계 방문지 같은 상품 구성 근거
- 웰니스, 의료, 생태, 도보 여행처럼 특정 테마에 필요한 전문 속성
- 외국어 스토리텔링, 오디오 가이드, 문화 해설 맥락

## 보강 원칙

1. 요청 의도와 현재 근거를 먼저 분석한다.
2. 필요한 API만 호출한다.
3. API 응답은 `source_documents`와 별도 signal table에 나누어 저장한다.
4. 생성 결과에는 출처, 조회 시각, 해석 주의사항을 함께 남긴다.
5. 방문자 수, 집중률, 내비게이션 연관성은 운영 판단 보조 지표로만 사용한다.
6. 가격, 예약 가능 여부, 의료 효과, 안전 보장 같은 항목은 API 응답만으로 확정하지 않는다.

## API별 활용 계획

### 1. 국문 관광정보 서비스_GW

공식 페이지:

- https://www.data.go.kr/data/15101578/openapi.do

현재 시스템의 기본 데이터 소스입니다. 현재는 목록/검색 중심 operation만 쓰고 있으므로, 가장 먼저 상세 계열 operation을 구현해야 합니다.

추가로 써야 할 operation:

| operation 후보 | 필요한 이유 |
|---|---|
| `detailCommon2` | 주소, 개요, 홈페이지, 좌표, 대표 이미지 등 기본 상세 보강 |
| `detailIntro2` | content type별 소개 속성 보강 |
| `detailInfo2` | 이용 시간, 주차, 요금, 문의 등 반복 정보 보강 |
| `detailImage2` | 상세 이미지 후보 보강 |
| `locationBasedList2` | 특정 관광지 주변 후보 탐색 |
| `categoryCode2` | 테마/콘텐츠 유형 기반 검색 정교화 |

활용:

- 상품 후보의 기본 설명을 길고 정확하게 만든다.
- QA에서 운영 시간, 휴무, 요금, 이용 조건의 누락 여부를 판정한다.
- Poster Studio에서 대표 이미지 후보를 먼저 확보한다.
- 장소 간 거리를 계산해 코스 구성의 현실성을 검토한다.

우선순위:

- P0: `detailCommon2`, `detailIntro2`, `detailInfo2`, `detailImage2`
- P1: `locationBasedList2`, `categoryCode2`

### 2. 관광공모전 사진 수상작 정보

공식 페이지:

- https://www.data.go.kr/data/15145706/openapi.do

공공데이터포털 설명 기준으로 포토코리아 관광공모전 사진 부문 수상작의 제목, 촬영일, 촬영지, 키워드, 이미지 정보를 제공합니다. 개방 데이터는 법정동코드 정보, 수상작 정보, 수상작 동기화 정보로 구성됩니다.

활용:

- 포스터/상세페이지의 시각 방향 후보를 만든다.
- 지역/계절/풍경 키워드 기반으로 이미지 후보를 추천한다.
- `Poster Prompt Agent`가 "어떤 분위기의 관광 포스터가 맞는지"를 판단할 때 참고한다.
- 사용자가 포스터 생성 옵션을 고를 때 참고 이미지 후보로 제공한다.

호출 조건:

- Run Review에서 특정 상품이 승인 또는 검토 가능 상태로 들어왔을 때
- 상품에 대표 이미지가 없거나 `detailImage2` 결과가 약할 때
- 포스터 생성, SNS 이미지, 상세페이지 히어로 이미지가 필요한 경우
- 지역명, 법정동, 촬영지, 키워드가 상품 후보와 매칭될 때

저장 후보:

```json
{
  "source_family": "kto_photo_contest",
  "title": "사진 제목",
  "shooting_date": "YYYY-MM-DD",
  "shooting_place": "촬영지",
  "keywords": ["야경", "바다", "축제"],
  "image_url": "https://...",
  "legal_dong_code": "...",
  "license_note": "공식 페이지 이용 조건 확인",
  "retrieved_at": "2026-05-06T00:00:00+09:00"
}
```

주의:

- 포스터 생성용 참고 이미지와 실제 게시 이미지 사용은 구분한다.
- 이미지 사용 전 이용허락범위, 원 출처, 인물/상표/시설물 제한을 확인한다.

### 3. 웰니스관광정보

공식 페이지:

- https://www.data.go.kr/data/15144030/openapi.do

공공데이터포털 설명 기준으로 웰니스 관광 정보는 법정동 코드, 지역기반 관광정보, 위치기반 관광정보, 키워드 검색, 동기화 정보, 공통정보, 소개정보, 반복정보, 이미지정보를 제공합니다. 여러 언어 제공이 가능하다는 점도 상품화에 중요합니다.

활용:

- 힐링, 명상, 스파, 자연치유, 뷰티, 푸드, 스테이 상품 후보를 찾는다.
- 고연령층, 프리미엄, 장기 체류, 회복형 여행 같은 타깃에 맞는 근거를 보강한다.
- 상품 카피에서 편안함, 회복, 휴식 같은 메시지를 근거 기반으로 다듬는다.
- 숙박/체험/식음 요소를 묶은 웰니스 패키지를 구성한다.

호출 조건:

- `preferred_themes`에 웰니스, 힐링, 명상, 스파, 뷰티, 자연치유, 휴식, 프리미엄이 포함될 때
- 타깃이 시니어, 커플, 여성 고객, 장기 체류 고객, 회복 목적 여행객일 때
- 일반 TourAPI 후보의 개요가 짧고 웰니스 속성 판단이 어려울 때

보강할 판단:

- 체험 성격: 휴식형, 치유형, 자연형, 뷰티형, 식음형
- 이용 조건: 예약, 이용 시간, 휴무, 주차, 시설 안내
- 이미지: 조용한 공간, 자연, 실내 스파, 음식 등 포스터 소재
- 외국어 대응 가능성: 여러 언어 데이터가 있을 경우 해외 고객용 카피 근거

주의:

- 건강 개선, 치료, 효능을 확정 표현하지 않는다.
- 의료관광정보와 섞일 때는 의료 광고성 표현 검수를 강화한다.

### 4. 의료관광정보

공식 페이지:

- https://www.data.go.kr/data/15143913/openapi.do

공공데이터포털 설명 기준으로 의료 관광 정보는 법정동 코드, 지역기반 관광정보, 위치기반 관광정보, 키워드 검색, 동기화 정보, 공통정보, 소개정보, 의료 관광정보를 제공합니다. 여러 언어 제공이 가능하다는 점을 외국인 대상 상품 검토에 활용할 수 있습니다.

활용:

- 외국인 대상 K-의료, 검진, 뷰티, 회복형 체류 상품 후보를 검토한다.
- 의료기관 방문 전후의 숙박, 음식, 관광 동선을 분리해 설계한다.
- 의료 서비스 자체와 주변 관광 상품을 구분해 QA 리스크를 관리한다.
- 다국어 안내가 필요한 상품의 설명 구조를 잡는다.

호출 조건:

- 요청에 의료, 병원, 검진, 뷰티, 성형, 피부, K-의료, 외국인 의료 방문이 포함될 때
- 웰니스 상품에서 의료적 표현이 섞일 가능성이 있을 때
- 상품 설명에 의료기관, 시술, 진료, 회복 기간, 부작용 같은 표현이 등장할 때

보강할 판단:

- 의료기관/의료관광 자원 여부
- 외국인 대상 설명 가능성
- 주변 숙박/이동/식사 동선과 분리해야 할 부분
- QA에서 의료 효능, 결과 보장, 안전 보장 표현 차단

주의:

- 의료 효과, 치료 결과, 시술 안전성, 가격을 생성 결과에서 확정하지 않는다.
- 의료기관 공식 정보와 현행 의료광고 규정 확인을 운영자 필수 검토로 남긴다.

### 5. 반려동물 동반여행 서비스

공식 페이지:

- https://www.data.go.kr/data/15135102/openapi.do

공공데이터포털 설명 기준으로 전국 관광지, 숙소, 음식점, 쇼핑시설 등 반려동물 동반 가능 장소를 제공합니다. 기본 정보와 함께 반려동물 동반 조건, 유의사항, 이용 가능 시설을 제공합니다.

활용:

- 반려동물 동반 여행 상품을 구성한다.
- 동반 가능 장소, 숙박, 음식점, 쇼핑시설을 하나의 운영 동선으로 묶는다.
- FAQ에 리드줄, 케이지, 체중 제한, 실내 출입 가능 범위, 예방접종 증빙 등 확인 항목을 넣는다.
- QA에서 반려동물 가능 여부를 과하게 표현하지 않도록 검수한다.

호출 조건:

- 요청에 반려동물, 반려견, 강아지, 펫프렌들리, 애견 동반이 포함될 때
- 가족/커플 상품에서 반려동물 동반 옵션을 추가하려 할 때
- 기존 상품 후보에 음식점/숙박이 포함되고 동반 조건 확인이 필요한 때

보강할 판단:

- 장소별 반려동물 동반 가능 조건
- 출입 제한 구역과 유의사항
- 이용 가능 시설
- 운영 시간/휴무일
- 반려동물 동반 고객에게 추가로 필요한 FAQ

주의:

- "반려동물 동반 가능" 표현은 조건과 함께 표시한다.
- 시설 정책은 바뀔 수 있으므로 게시 전 재확인을 요구한다.

### 6. 두루누비 정보 서비스_GW

공식 페이지:

- https://www.data.go.kr/data/15101974/openapi.do

공공데이터포털 설명 기준으로 코리아둘레길 284개 코스의 상세 GPX 정보와 주변 관광정보를 제공합니다. 제공 operation은 코스 목록 정보 조회와 길 목록 정보 조회입니다.

활용:

- 도보, 트레킹, 로컬 산책, 장거리 걷기 상품을 구성한다.
- 코스 거리, 예상 소요 시간, 경유지, GPX를 운영 근거로 저장한다.
- 상품 페이지에 코스 지도, 난이도, 집결/해산 지점, 안전 유의사항을 넣는다.
- Poster Studio에서 "걷기 여행", "해안길", "둘레길" 같은 시각 키워드를 강화한다.

호출 조건:

- 요청에 걷기, 트레킹, 둘레길, 산책, 해안길, 코스, GPX가 포함될 때
- 상품 후보 itinerary가 도보 이동을 포함하지만 거리/난이도 근거가 없을 때
- 액티비티 상품의 안전/체력 리스크 검수가 필요한 때

보강할 판단:

- 코스 길이와 구간
- GPX 또는 route asset
- 주변 관광지 조합
- 시작/종료 지점
- 체력, 날씨, 교통 접근성 리스크

주의:

- GPX는 실제 현장 운영 동선과 다를 수 있으므로 운영자가 최종 경로를 확인한다.
- 악천후, 야간, 어린이/시니어 동반 상품에서는 안전 문구를 강화한다.

### 7. 관광지 오디오 가이드정보_GW

공식 페이지:

- https://www.data.go.kr/data/15101971/openapi.do

공공데이터포털 설명 기준으로 오디(Odii)의 관광지 음성, 대본, 사진 정보를 제공합니다. 관광지 기본 정보, 관광지 및 이야기 위치 기반 정보, 키워드 검색 목록, 동기화 목록 등이 있으며 한국어, 영어, 중국어, 일본어 이야기를 제공합니다.

활용:

- 역사, 문화, 로컬 스토리 중심 상품의 설명 밀도를 높인다.
- 외국인 대상 상품에 다국어 해설 가능성을 반영한다.
- 상세페이지 카피와 FAQ에서 스토리텔링 근거를 만든다.
- 오디오 가이드가 있는 장소를 itinerary에 우선 반영한다.

호출 조건:

- 요청에 역사, 문화, 유산, 골목, 로컬 스토리, 해설, 외국인, 다국어가 포함될 때
- 상품 후보에 궁궐, 박물관, 문화재, 구도심, 전통시장 등이 포함될 때

보강할 판단:

- 장소별 이야기 주제
- 지원 언어
- 오디오/대본/사진 asset
- 관광지와 이야기 좌표

주의:

- 대본 원문을 장문 복제하지 않고 요약과 출처 중심으로 저장한다.

## 데이터 공백 분류

Data Agent는 현재 source item과 retrieved document를 보고 다음 공백을 구조화해야 합니다.

```json
[
  {
    "gap_type": "missing_detail_info",
    "severity": "high",
    "required_for": ["product_generation", "qa_review"],
    "candidate_sources": ["kto_tour_detail_info"],
    "reason": "상품 운영 시간과 휴무일이 없어 게시 전 검토가 어렵습니다."
  },
  {
    "gap_type": "missing_visual_asset",
    "severity": "medium",
    "required_for": ["poster_studio", "marketing_assets"],
    "candidate_sources": ["kto_tourism_photo", "kto_photo_contest"],
    "reason": "대표 이미지와 포스터 참고 이미지가 부족합니다."
  }
]
```

공백 유형:

| gap_type | 의미 | 우선 source |
|---|---|---|
| `missing_detail_common` | 상세 주소/홈페이지/개요 부족 | 국문 관광정보 상세 |
| `missing_detail_info` | 이용 시간, 요금, 주차, 휴무 부족 | 국문 관광정보 반복정보 |
| `missing_image_asset` | 대표/상세 이미지 부족 | 상세 이미지, 관광사진, 공모전 사진 |
| `missing_pet_policy` | 반려동물 조건 부족 | 반려동물 동반여행 |
| `missing_wellness_attributes` | 웰니스 성격 판단 부족 | 웰니스관광정보 |
| `missing_medical_context` | 의료관광 판단/검수 근거 부족 | 의료관광정보 |
| `missing_route_asset` | 도보 코스/거리/GPX 부족 | 두루누비 |
| `missing_story_asset` | 문화/역사 해설 부족 | 오디오 가이드 |
| `missing_sustainability_context` | 생태/공정관광 맥락 부족 | 생태 관광 |
| `missing_demand_signal` | 지역 수요 근거 부족 | 관광빅데이터 |
| `missing_crowding_signal` | 일정별 혼잡 판단 부족 | 집중률 예측 |
| `missing_related_places` | 주변 코스 조합 근거 부족 | 연관 관광지 |
| `missing_user_business_info` | 가격/파트너/예약/집결지/취소 정책 부족 | 공식 웹 근거 검색, 사용자 입력, 내부 DB |

`missing_user_business_info`는 바로 사용자 질문으로 넘기지 않습니다. 공식 홈페이지, 공식 예약 페이지, 행사 공지, 운영사 정책 페이지에서 먼저 확인하고, 그래도 확정할 수 없는 항목만 사용자 입력 또는 내부 DB 확인 대상으로 남깁니다. 파트너 정산 조건, 최종 판매가, 플랫폼 자체 취소/환불 정책처럼 외부 웹에 공개되지 않는 정보는 사용자/내부 DB를 우선합니다.

## 우선 구현 순서

### Phase 1. 현재 TourAPI 상세 보강

목표:

- 현재 `TourApiProvider`를 확장해 `detailCommon2`, `detailIntro2`, `detailInfo2`, `detailImage2`를 추가한다.
- 기존 `tourism_items.raw`와 `source_documents`에 상세/반복/이미지 정보를 반영한다.
- QA에서 "운영자가 확인해야 할 항목"과 "API로 확인된 항목"을 구분한다.

현재 코드 반영:

- 프로젝트 Phase 9에서 구현 완료.
- `detailCommon2`, `detailIntro2`, `detailInfo2`, `detailImage2`는 workflow Data 단계와 상세 보강 API에 연결되었습니다.
- `categoryCode2`, `locationBasedList2`는 provider method와 capability catalog에 추가되었지만, 실제 route/ranking workflow 연결은 이후 단계에서 진행합니다.
- content_id 기반 상세 보강 API는 `POST /api/data/tourism/details/enrich`입니다.
- 기존 검색 API는 `GET /api/data/tourism/search?enrich_details=true`일 때 상세 보강까지 수행합니다.
- 상세 이미지 결과는 `tourism_visual_assets.usage_status=candidate`로 저장합니다.
- 보강된 상세/반복/이미지 metadata는 `source_documents`와 Chroma index에 반영합니다.

이 Phase를 먼저 하는 이유:

- 기존 DB 모델과 RAG 구조에 가장 자연스럽게 붙는다.
- 현재 상품 초안의 근거 밀도를 즉시 높인다.
- 이후 테마 API를 붙이기 전 공통 상세 구조를 정할 수 있다.

### Phase 2. 시각 자료와 코스 조합 보강

대상:

- 관광사진 정보
- 관광공모전 사진 수상작 정보
- 관광지별 연관 관광지 정보
- 관광지 집중률 방문자 추이 예측 정보

목표:

- Poster Studio 준비 데이터를 축적한다.
- 상품별 주변 관광지/음식/숙박 조합을 보강한다.
- 혼잡 리스크를 QA와 운영 메모에 반영한다.

### Phase 3. 테마 특화 API 보강

대상:

- 웰니스관광정보
- 반려동물 동반여행 서비스
- 두루누비
- 오디오 가이드
- 생태 관광

목표:

- 상품 유형에 따라 필요한 API를 선택 호출한다.
- 테마별 전문 속성과 주의사항을 상품/FAQ/QA에 반영한다.

### Phase 4. 의료관광정보 보강

목표:

- 외국인 의료관광/웰니스 경계의 상품 후보를 다룬다.
- 의료 광고성 표현, 효능 보장, 가격 단정 표현을 QA에서 강하게 제한한다.
- 의료기관 공식 확인 필요 항목을 별도 검토 필드로 둔다.

의료관광은 리스크가 높으므로 테마 API 중 가장 늦게 붙이는 편이 좋습니다.

## 생성 결과에 반영할 데이터 표현

좋은 표현:

- "공식 관광정보와 연관 관광지 데이터를 기준으로 주변 음식/숙박 후보를 함께 검토했습니다."
- "이 장소는 향후 30일 집중률 예측을 별도 확인한 뒤 운영 날짜를 확정하는 편이 좋습니다."
- "반려동물 동반 조건은 장소별로 다르므로 상품 FAQ에 제한 조건을 명시해야 합니다."
- "포스터 제작에는 포토코리아 이미지 키워드와 상품 리뷰 내용을 함께 사용합니다."
